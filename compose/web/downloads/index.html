● Suricatoos Agent · v0.1.0

Downloads do Agente

Agente de endpoint passivo e local: inventaria pacotes e SO da máquina e reporta outbound (mTLS) ao pipeline Suricatoos. A correlação roda no servidor e os achados aparecem no scanner. Não faz varredura de rede, não coleta arquivos, credenciais nem telemetria.

Linux .deb / .rpm macOS .pkg universal Windows .msi x64 Pacotes Linux assinados (GPG)
Antes de começar — para o agente se conectar você precisa de um token de enrollment e do CA-pin, gerados pelo administrador no control-plane (veja Para administradores). O servidor de enrollment é https://scanner.suricatoos.com/agent/v1.

1. Escolha o sistema operacional

Selecione a plataforma do endpoint para ver o passo a passo específico.

Debian / Ubuntu · x86_64.deb (amd64) Baixar .deb
Debian / Ubuntu · ARM64.deb (arm64) Baixar .deb
RHEL / Fedora / SUSE · x86_64.rpm (x86_64) Baixar .rpm
RHEL / Fedora / SUSE · aarch64.rpm (aarch64) Baixar .rpm

  1. Verifique a assinatura (recomendado)

    Os pacotes são assinados pela chave Suricatoos Agent Packages. Importe a chave pública e cheque o RPM:

    curl -LO https://github.com/williamsouzadelima/suricatoos-infra/releases/download/agent-v0.1.0/suricatoos-agent-pkg-pub.asc
sudo rpm --import suricatoos-agent-pkg-pub.asc
rpm -K suricatoos-agent-*.rpm        # -> "digests signatures OK"

    Confira também os checksums (SHA256SUMS): sha256sum -c SHA256SUMS

  2. Instale o pacote

    # Debian / Ubuntu
sudo apt install ./suricatoos-agent_0.1.0_amd64.deb
# RHEL / Fedora / SUSE
sudo dnf install ./suricatoos-agent-0.1.0-1.x86_64.rpm

    Instala /usr/bin/suricatoos-agent, a unit systemd (ainda não habilitada) e cria /var/lib/suricatoos-agent.

  3. Enrolle o agente

    Use o token e o CA-pin fornecidos pelo administrador:

    sudo suricatoos-agent enroll --state /var/lib/suricatoos-agent \
  --server https://scanner.suricatoos.com/agent/v1 \
  --token "<TOKEN>" --ca-pin "<CA-PIN>"

    A URL de ingest é herdada do enrollment — não precisa de --ingest.

  4. Habilite e inicie o serviço

    sudo systemctl enable --now suricatoos-agent
systemctl status suricatoos-agent
journalctl -u suricatoos-agent -f
Remover 
sudo apt remove suricatoos-agent     # ou: dnf remove suricatoos-agent
# o state (/var/lib/suricatoos-agent, com os certs) é preservado; apague com:
sudo rm -rf /var/lib/suricatoos-agent

  1. Confira o checksum

    Instala /usr/local/bin/suricatoos-agent, o LaunchDaemon (ainda não carregado) e cria /var/lib/suricatoos-agent.

  4. Carregue o serviço (launchd)

    sudo launchctl bootstrap system /Library/LaunchDaemons/com.suricatoos.agent.plist
sudo launchctl print system/com.suricatoos.agent | grep state
# logs: /var/log/suricatoos-agent.log

sudo rm -f /Library/LaunchDaemons/com.suricatoos.agent.plist /usr/local/bin/suricatoos-agent
sudo pkgutil --forget com.suricatoos.agent
# state (certs) preservado; apague com: sudo rm -rf /var/lib/suricatoos-agent
Windows · x64.msi (x64) Baixar .msi
O MSI atual é não assinado — o SmartScreen/antivírus pode avisar. Em produção, assine com Authenticode (signtool).

  1. Instale o MSI

    msiexec /i suricatoos-agent-0.1.0.msi /qn

    Instala suricatoos-agent.exe em C:\Program Files\Suricatoos Agent\.

    cd "C:\Program Files\Suricatoos Agent"
.\suricatoos-agent.exe enroll --state "C:\ProgramData\Suricatoos\agent" `
  --server https://scanner.suricatoos.com/agent/v1 `
  --token "<TOKEN>" --ca-pin "<CA-PIN>"

    .\suricatoos-agent.exe install      # serviço "SuricatoosAgent", start automático

    State (certs) em C:\ProgramData\Suricatoos\agent.

Remover 
.\suricatoos-agent.exe uninstall   # remove o serviço SCM
msiexec /x suricatoos-agent-0.1.0.msi /qn
# state preservado; apague com: rmdir /s "C:\ProgramData\Suricatoos\agent"

3. Para administradores — gerar um token

O token e o CA-pin saem do control-plane. O comando abaixo gera um bundle de enrollment (YAML) já com server, token e ca_pin prontos para entregar ao operador.

curl -sS -X POST https://scanner.suricatoos.com/agent/api/v1/tokens \
  -H "Authorization: Bearer <ADMIN_SECRET>" \
  -H "Content-Type: application/json" \
  -d '{"type":"single_host","tenant":"default","ttl_hours":24}' \
  -OJ        # salva enroll-<id>.yaml

Tipos: single_host (1 uso) ou deployment (vários hosts, use "max_uses"). Liste com GET /agent/api/v1/tokens e revogue com DELETE /agent/api/v1/tokens/{id} (revogar também invalida os certificados mTLS emitidos por aquele token). O <ADMIN_SECRET> é secreto — nunca o coloque em scripts versionados ou nesta página.
